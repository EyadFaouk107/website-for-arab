# Arab Contractors — Corporate Website

Official website for Arab Contractors (Osman Ahmed Osman & Co.).

## Project Structure

```
arab-contractors/
├── index.html              ← Homepage
├── about.html
├── projects.html
├── project-detail.html
├── sectors.html
├── innovation.html
├── careers.html
├── contact.html
├── news.html
│
├── vercel.json             ← Clean URL rewrites
├── vite.config.ts          ← Multi-page Vite build
│
└── src/
    ├── main.tsx            ← Vite entry point (imports CSS + JS)
    ├── main.css            ← Master CSS (imports all partials in order)
    │
    ├── css/
    │   ├── tokens.css      ← Design tokens (colors, spacing, radii, shadows…)
    │   ├── base.css        ← Reset, body, headings, .overlay
    │   ├── layout.css      ← Container, grid, flex, section variants, spacing utils
    │   ├── components.css  ← Header, nav, drawer, hero, buttons, cards, forms, footer…
    │   ├── animations.css  ← Keyframes, .reveal system, microinteractions, GT suppression
    │   └── utilities.css   ← Text, color, opacity, border-radius, shadow, responsive utils
    │
    └── js/
        ├── main.js         ← Entry: imports & calls all module init functions
        ├── partials.js     ← Shared header/drawer/footer HTML (eliminates duplication)
        └── modules/
            ├── navigation.js   ← Sticky header, mobile drawer, mega menu, active link
            ├── animations.js   ← IntersectionObserver scroll-reveal
            ├── counters.js     ← Animated number counters (data-count attribute)
            ├── slider.js       ← Auto-advancing hero slider, pause on hover
            ├── projects.js     ← Filter buttons + live search + ?sector= URL param
            ├── forms.js        ← Contact form validation + submission feedback
            ├── youtube.js      ← RSS feed fetch → video cards, graceful fallback
            └── translation.js  ← EN ↔ AR toggle via Google Translate widget API
```

## Getting Started

### Prerequisites
- Node.js 18+

### Install & Run

```bash
npm install
npm run dev        # http://localhost:3000
```

### Build for Production

```bash
npm run build      # outputs to /dist
npm run preview    # preview the production build
```

## Key Design Decisions

### CSS Architecture
All design tokens (colors, spacing, shadows, transitions, typography, z-index) live in a single `tokens.css` file.
CSS files are imported in strict cascade order: `tokens → base → layout → components → animations → utilities`.
No conflicting definitions, no duplicated variables.

### JavaScript Modules
Each module is:
- **Self-contained** — guards against missing DOM nodes, never throws if a widget isn't on the page.
- **Side-effect free** — only runs when its `init*` function is explicitly called from `main.js`.
- **Named with clear intent** — `initNavigation`, `initCounters`, etc.

### Shared Partials
Header, mobile drawer, and footer HTML are generated by `partials.js` and injected via `renderHeader()`, `renderDrawer()`, and `renderFooter()`.
This eliminates copy-paste across all 9 pages. To update navigation, edit **one file**.

### Scroll Reveal System
The old class names (`reveal-up`, `reveal-left`, `reveal-right`) are replaced with BEM modifiers
(`reveal--up`, `reveal--left`, `reveal--right`) and the active class changed from `.active` to `.is-visible`.
Stagger delays are driven by `data-delay="1..5"` attributes rather than inline `style="transition-delay"`.

### Counter Attributes
Old: `data-target` + `data-suffix` on `.stat-number`
New: `data-count` + `data-suffix` on any element — cleaner and more semantic.

### Translation
Uses the Google Translate widget combo-select API instead of cookie + page reload.
Language choice persists across pages via `localStorage`.
